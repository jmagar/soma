# Soma Code Mode And OpenAPI Port Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Port Lab's `labby-openapi` and `labby-codemode` into Soma as self-contained `soma-openapi` and `soma-codemode` crates with complete behavior parity, stronger documented hardening where specified, and no dependency on Lab or existing Soma crates.

**Architecture:** Create two standalone workspace crates. `soma-openapi` owns hardened OpenAPI config, parsing, registry, SSRF policy, and HTTP dispatch with only external dependencies. `soma-codemode` owns the Code Mode runner, protocol, local providers, support primitives, and an optional `openapi` feature that is the only path to `soma-openapi`.

**Tech Stack:** Rust 2021, rust-version 1.96, Cargo resolver 2, Tokio, reqwest/rustls, serde/serde_json, Javy/QuickJS, tempfile, wiremock, Beads (`bd`), Soma `xtask` gates.
